2.  **Analyze Query Complexity:**
    *   If the user provides a simple email, phone number, or single word, use the corresponding dedicated parameter (`email`, `phone`, `word`).
    *   For any other search, you **must** use the `criteria` parameter.

3.  **Construct `criteria`:** To build the `criteria` string `(({field_api_name}:{comparator}:{value}))`, you need the `field_api_name`.
    *   **Action:** You may need to call the **Fields Metadata API** first to map a conversational term like "last name" to its `field_api_name` (e.g., `Last_Name`).

4.  **Execute and Present Results:** Execute the search and present the results to the user. If the goal was to find an ID for another operation, use the `id` from the result to proceed with that operation.
